# Checklist contrato piloto

## Condicion principal

No lanzar piloto comercial apoyado en el modelo base sin resolver permisos comerciales o sustituir el motor.

## Debe cubrir

- Alcance exacto del piloto.
- Numero de assets incluidos.
- Duracion maxima de piezas.
- Entregables: dashboard, PDF, workshop, reanalisis.
- Precio y forma de pago si aplica.
- Uso permitido: decision creativa sobre contenidos.
- Uso prohibido: biometria, audiencia real, empleados, estudiantes, emocion individual, medical decision support.
- Derechos sobre assets aportados por cliente.
- Confidencialidad.
- Retencion y borrado.
- Subprocessors.
- No entrenamiento con assets salvo opt-in.
- Limitaciones metodologicas.
- No garantia de ventas, ROI, CTR, recuerdo o conducta.
- Responsabilidad humana final del cliente.

## Anexos recomendados

- DPA.
- Politica de retencion.
- Ficha de seguridad.
- Lista de subprocessors.
- Descripcion tecnica de pipeline.
- Guia de lenguaje y limites.
