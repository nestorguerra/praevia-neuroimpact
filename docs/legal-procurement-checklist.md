# Checklist procurement

## Producto

- [ ] Descripcion del servicio.
- [ ] Alcance del piloto.
- [ ] Entregables: dashboard, informe, workshop, export.
- [ ] Responsable PraevIA y responsable cliente.
- [ ] Limites cientificos aceptados.

## Legal

- [ ] DPA revisado.
- [ ] Politica de privacidad revisada.
- [ ] Terminos de servicio revisados.
- [ ] Contrato piloto revisado.
- [ ] Clearance TRIBE o alternativa aprobado.
- [ ] Subprocesadores anexados.
- [ ] Region/transferencias documentadas.

## Seguridad

- [ ] Auth real.
- [ ] Roles y permisos.
- [ ] RLS por organizacion.
- [ ] Storage privado.
- [ ] URLs firmadas.
- [ ] Secretos fuera del frontend.
- [ ] Logs de auditoria.
- [ ] Borrado seguro.
- [ ] Backups.
- [ ] Observabilidad.

## Datos

- [ ] No se suben datos biometricos.
- [ ] No se suben grabaciones de audiencia real.
- [ ] Retencion acordada.
- [ ] Procedimiento de borrado acordado.
- [ ] Uso de assets para entrenamiento prohibido salvo permiso expreso.

## Operacion

- [ ] Limite de creditos.
- [ ] Cost cap.
- [ ] Duracion maxima de assets.
- [ ] SLA piloto.
- [ ] Soporte y escalado.
- [ ] Offboarding.

## Comercial

- [ ] Precio y alcance cerrados.
- [ ] Facturacion manual beta.
- [ ] Sin pasarela de pago.
- [ ] No promesas de resultado de negocio.
- [ ] Claims revisados en landing, app, PDF y deck.
